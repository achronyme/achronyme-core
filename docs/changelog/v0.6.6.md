# Version 0.6.6 - Native GUI & System I/O

This release transforms Achronyme from a pure computational engine into a full-stack application platform. It introduces a high-performance Native GUI system based on immediate mode rendering (egui), alongside comprehensive system capabilities including File I/O, Networking, and Environment management.

## üñ•Ô∏è Native GUI System

Achronyme now supports building cross-platform desktop applications directly in Achronyme code. The GUI system is reactive, thread-safe, and hardware-accelerated.

### Core Functions

| Function | Signature | Description |
|---|---|---|
| `gui_run` | `(render_fn: Function, options?: Record)` | Starts the GUI application loop. Blocking call. |

### UI Components

| Component | Signature | Description |
|---|---|---|
| `ui_label` | `(text: String, style?: String)` | Displays text. |
| `ui_button` | `(text: String, style?: String) -> Boolean` | Clickable button. Returns true if clicked. |
| `ui_text_input` | `(signal: Signal, style?: String) -> Boolean` | Editable text field bound to a signal. |
| `ui_slider` | `(signal: Signal, min: Number, max: Number, style?: String)` | Numeric slider control. |
| `ui_checkbox` | `(signal: Signal, label: String, style?: String)` | Boolean toggle. |
| `ui_combobox` | `(signal: Signal, options: Vector, style?: String)` | Dropdown selection. |
| `ui_radio` | `(signal: Signal, value: Any, label: String, style?: String)` | Radio button for exclusive selection. |
| `ui_progress_bar` | `(value: Number, style?: String)` | Progress indicator (0.0 to 1.0). |
| `ui_separator` | `(style?: String)` | Visual divider. |

## üåê System & Networking

Capabilities to interact with the operating system and the web.

### File I/O (Async)

All file operations are asynchronous and return a Future.

| Function | Signature | Description |
|---|---|---|
| `read_file` | `(path: String) -> Future<String>` | Reads file content. |
| `write_file` | `(path: String, content: String) -> Future<Null>` | Writes content (overwrites). |
| `append_file` | `(path: String, content: String) -> Future<Null>` | Appends content. |
| `delete_file` | `(path: String) -> Future<Boolean>` | Deletes a file. |
| `exists` | `(path: String) -> Future<Boolean>` | Checks file existence. |

### Networking (Async)

| Function | Signature | Description |
|---|---|---|
| `http_get` | `(url: String) -> Future<String>` | Performs HTTP GET. |
| `http_post` | `(url: String, body: String, headers?: Record) -> Future<String>` | Performs HTTP POST. |

### Environment Variables

| Function | Signature | Description |
|---|---|---|
| `env_get` | `(key: String) -> String\|Null` | Gets an environment variable. |
| `env_set` | `(key: String, value: String) -> Null` | Sets an environment variable. |
| `env_vars` | `() -> Record` | Returns all variables. |
| `env_load` | `(path?: String) -> Boolean` | Loads variables from a `.env` file. |

### Data Encoding

| Function | Signature | Description |
|---|---|---|
| `json_parse` | `(json: String) -> Value` | Parses JSON string. |
| `json_stringify` | `(value: Any, pretty?: Boolean) -> String` | Serializes value to JSON. |
| `csv_parse` | `(csv: String, headers?: Boolean) -> Vector` | Parses CSV string. |
